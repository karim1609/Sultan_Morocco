\chapter{Conclusion generale}
\label{ch:conclusion}

\section{Synthese}

Sultan Morocco demontre qu'une experience touristique riche peut etre livree avec un \textbf{monolithe Laravel maitrise}, sans dependance couteuse a des APIs de listing. Les choix statiques + enrichissement OSM + chatbot hybride constituent un equilibre pertinent pour un MVP oriente demonstration et iteration rapide.

\section{Criteres de succes atteints}

\begin{itemize}
  \item Parcours visiteur complet (accueil $\rightarrow$ hotels/restaurants/carte).
  \item Comptes utilisateurs et messagerie operationnels.
  \item Documentation technique (README + present rapport).
  \item Pipeline install reproductible (\texttt{composer run setup}).
\end{itemize}

\section{Travaux restants prioritaires}

\begin{enumerate}
  \item Back-office ou CMS pour listings.
  \item Suite de tests Feature + pipeline CI.
  \item Durcissement securite profils et chatbot.
  \item Activation progressive du schema \texttt{places}.
\end{enumerate}
