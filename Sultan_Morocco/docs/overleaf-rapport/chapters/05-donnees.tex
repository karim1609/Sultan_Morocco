\chapter{Modele de donnees et persistance}
\label{ch:data}

\section{Strategie de persistance}

L'application adopte un modele \textbf{hybride} : donnees touristiques en memoire PHP (read-only a l'execution) et donnees sociales/compte en base relationnelle. Ce decouplage isole la volumetrie future des POI du trafic transactionnel utilisateur.

\section{Configuration par defaut}

\begin{itemize}
  \item \textbf{SQLite} : fichier \texttt{database/database.sqlite}
  \item \texttt{SESSION\_DRIVER=database}
  \item \texttt{CACHE\_STORE=database}
  \item \texttt{QUEUE\_CONNECTION=database}
\end{itemize}

\section{Tables actives}

\subsection{Utilisateurs et sessions}

Table \texttt{users} : \texttt{name}, \texttt{email}, \texttt{password} (bcrypt), \texttt{avatar}, \texttt{role} (enum \texttt{user|admin|author}), \texttt{country}, \texttt{email\_verified\_at}.

\subsection{Messagerie}

\begin{infobox}{Algorithme findOrCreateDirectBetween}
\begin{enumerate}
  \item Si $A = B$ : erreur 403 (impossible de se parler \`a soi-m\^eme).
  \item Chercher une conversation existante avec exactement les participants $A$ et $B$.
  \item Si trouv\'ee : retourner cette conversation.
  \item Sinon : cr\'eer \texttt{conversations}, attacher $A$ et $B$ sur \texttt{conversation\_user}.
\end{enumerate}
\end{infobox}

Champs \texttt{messages} : \texttt{body} (texte), \texttt{read\_at} (nullable timestamp).

\section{Seeder de démonstration}

\texttt{DatabaseSeeder} crée \textbf{200 utilisateurs} via \texttt{UserFactory} pour tester profils et messagerie. Aucune donnée de lieux n'est injectée.
